Frontend

App() ⇒ JSX.Element​

Main App component that sets up the routing for the application. The app is wrapped with the WebSocketProvider to allow WebSocket context access throughout the app.

Kind: global function
Returns: JSX.Element - The App component JSX, which contains the routes for the game pages.

Night() ⇒ JSX.Element​

Night phase component for Mafia game. This component handles voting, player actions, and WebSocket communication for the night phase of the game.

Kind: global function
Returns: JSX.Element - The rendered JSX element for the night phase, including role display, voting section, and player lists.

useWebSocket ⇒ WebSocket | null​

Custom hook to access the WebSocket instance from the context.

Kind: global constant
Returns: WebSocket | null - The WebSocket instance or null if not connected.

WebSocketProvider ⇒ JSX.Element​

WebSocketProvider component that manages the WebSocket connection and provides it to the app context. It listens for events like open, message, error, and close to manage WebSocket behavior.

Kind: global constant
Returns: JSX.Element - The WebSocket provider component.

Component:

ParamTypeDescription
propsObjectThe component's props.
props.childrenReact.ReactNodeThe child components that will consume the WebSocket context.

    • ~wsRef : React.RefObject.<WebSocket>
      • .current.onopen() ⇒ void
      • .current.onmessage(event) ⇒ void
      • .current.onerror(error) ⇒ void
      • .current.onclose(event) ⇒ void

GameOver() ⇒ JSX.Element​

GameOver component to display the game-over message and play the corresponding sound based on the winner.

Kind: global function
Returns: JSX.Element - - The rendered JSX element for the game over screen.
Component:

  • GameOver() ⇒ JSX.Element
    • ~speak(sound, callback)
    • ~audio()

GameOver~speak(sound, callback)​

Function to play an audio file and invoke a callback once playback finishes.

Kind: inner method of GameOver

ParamTypeDescription
soundstringThe sound file path to be played.
callbackfunctionThe callback function to be executed after the audio finishes playing.

GameOver~audio()​

Function to handle the sequence of sounds to be played based on the winner. It plays a sequence of sounds: Game Over sound followed by either the C win sound or M win sound.

Kind: inner method of GameOver

Eliminated() ⇒ JSX.Element​

Eliminated component handles the elimination phase of the game. It plays sounds, displays the elimination message, and navigates between game phases (Day and Night).

Kind: global function
Returns: JSX.Element - - The rendered JSX element for the elimination screen.
Component:

  • Eliminated() ⇒ JSX.Element
    • ~audio() ⇒ void
    • ~speak(sound, callback) ⇒ void

Eliminated~audio() ⇒ void​

Handles the playing of multiple sounds in sequence. Plays the "End Vote" sound first, then based on the elimination status, it plays either the "Elim" or "NoElim" sound.

Kind: inner method of Eliminated

Eliminated~speak(sound, callback) ⇒ void​

Plays an audio file and executes a callback function once the audio finishes playing.

Kind: inner method of Eliminated

ParamTypeDescription
soundstringThe sound file to be played.
callbackfunctionThe callback function to be executed after the sound finishes playing.

Dead() ⇒ JSX.Element​

Dead component displays the "You Are Dead" message to players who have been eliminated. It listens for a gameOver message from the WebSocket connection and navigates to the GameOver screen when the game is finished.

Kind: global function
Returns: JSX.Element - - The rendered JSX element for the dead screen.

index​

Main entry point for the React application.

This code renders the main <App /> component into the root DOM element and sets up performance reporting for the app.

Requires: module:React, module:ReactDOM, module:App, module:reportWebVitals

index~root : ReactDOM.Root​

The root DOM element for the application. This element is where the React app will be mounted.

Kind: inner constant of index
